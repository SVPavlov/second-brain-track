AI Second Brain Starter Kit

Your second brain, in plain files

A structured folder of markdown that an AI reads and writes for you. It speeds up your admin, remembers your client context, and thinks with you.

This is not a chatbot with memory. It is a set of plain text files you own, and an AI (Claude Code) operates on them. The files are the brain. The AI is the engine. That distinction is why you can start one before the AI is even set up, and why it is yours to keep and move anywhere.

What the kit gives you

  • A vault template: the folder structure, an operating manual, and a catalog.
  • Eleven skills you trigger in plain words (meeting notes, client prep, inbox to actions, and more).
  • A quality discipline (synthesis) that keeps the brain trustworthy as it grows.
  • A data rule and trust rules so it stays safe.

It runs today in Claude Code on your own laptop, opens in any editor, and moves to the enterprise stack later as a folder move, not a migration.

Quick Start

From zero to a working brain in five steps. Full detail per operating system is in Install.

  1. Install Claude Code. One line, no Node.js needed. 
    # macOS
curl -fsSL https://claude.ai/install.sh | bash
# Windows (PowerShell)
irm https://claude.ai/install.ps1 | iex
    Check it: claude --version.
  2. Get the kit. Download from placid-ripple-8nh2.here.now and unzip it.
  3. Make it a git repo (your version history and backup): 
    cp -r vault-template ~/my-brain
cd ~/my-brain
git init
git add -A
git commit -m "my brain"
  4. Start it and set up. Run claude, then say: 
    set up my second brain
    It interviews you, fills your profile, files your first document, and runs a first quality check.
  5. Back it up to GitHub. See Backup & Updates.
No Claude yet, or a locked-down laptop?

You can still begin. The files exist before any AI touches them. Fill your profile by hand or in claude.ai (free), and run the full thing once you can install Claude Code. Zero rework.

Install (Windows & macOS)

Written for someone who has never opened a terminal. A terminal is just a window where you type a command and press Enter.

1. Install Claude Code

macOS: open Terminal (press Cmd+Space, type Terminal, Enter), paste and run:

curl -fsSL https://claude.ai/install.sh | bash

Windows: open PowerShell (press Win+X, click Windows PowerShell), paste and run:

irm https://claude.ai/install.ps1 | iex

Verify with claude --version. If the installer is blocked, install Node.js LTS from nodejs.org and run npm install -g @anthropic-ai/claude-code instead.

2. Log in

Run claude. A browser opens; sign in with your Claude Pro or Max account. It remembers you after that.

3. Install git

macOS: run git --version; if missing, macOS offers to install it. Windows: install from git-scm.com (Next on every screen).

Stuck on a step?

Run claude doctor for a health check, and bring the exact error message to your group. Most fixes are one line.

The Vault Structure

Boring on purpose. The structure is what makes the AI reliable, not retrieval magic.

my-brain/
├── CLAUDE.md            the operating manual the AI follows
├── 00-START-HERE.md     the map and the catalog
├── 01-me.md             who you are
├── 02-command-center.md your daily cockpit
├── 03-tasks.md          your open loops
├── 04-log.md            what the brain did
├── 05-synthesis.md      the quality check + your questions
├── Inbox/               raw drops, emptied during ingest
├── Meetings/  People/   one note per meeting / person
├── Clients/  Projects/  one folder per client / project
├── Patterns/  Sources/  reusable plays / reference docs
└── Archive/             superseded material

Two layers: the numbered files (00 to 05) are working memory (always in view), and the capitalized folders are long-term memory. The two files the AI reads first are CLAUDE.md (the contract) and 00-START-HERE.md (the map it rebuilds after every change).

Conventions

  • Filenames in kebab-case; dated records as YYYY-MM-DD-short-title.md.
  • Four frontmatter fields on every note: type, tags, last_updated, related.
  • Wikilinks make the graph. A note nothing links to is a defect, not a preference.

Your Profile

01-me.md is who you are: your role, your clients (names and roles only), your recurring work, and your top time-sinks. The better this file is, the better your brain works, because the AI reads it on any task about your work.

You do not write it by hand. Say set up my second brain and the AI interviews you one question at a time, then writes the file. You can refine it anytime.

The time-sink list is the point

Naming your top three weekly admin time-sinks tells the brain what to win back first, and gives you a target to measure against.

Skills

Skills are capabilities you trigger in plain words. You ask for them; you do not hand-roll them. They are paste-based, so they work with zero connectors.

Say it likeWhat it does
set up my second brainGuided first-run: interview, first ingest, first synthesis
process my inboxFiles any document or pasted text in the right place, linked
I just had a meetingTurns notes or a transcript into a meeting note with actions
brief me on [client]Everything the brain knows about a client before a call
run synthesisThe quality pass (see below)
grill me before [event]The brain plays the toughest person in the room; you rehearse
save this as a patternStores a reusable play for the next engagement
daily briefRefreshes your command center
record this decision (or /decide)Captures what you chose, what you rejected and why, filed with the client
wrap up my session (or /wrap-up)End-of-day handoff: done / next / blocked, picked up by tomorrow's daily brief
paste an emailExtracts actions and drafts a reply in your voice

Skills are maintained centrally and shipped in the kit at .claude/skills/. You report problems; fixes reach you through updates.

How Memory Works

When you ingest something (process my inbox, or paste it), the brain reads it, decides where it lives, links it to related notes, and updates the catalog in 00-START-HERE.md. Nothing lands at the root.

Where things go

ContentHome
A meetingMeetings/
A personPeople/
Anything about a clientClients/<client>/
Non-client workProjects/
A reusable playPatterns/
Reference docsSources/
Action itemsappended to 03-tasks.md

Always verify where a document landed and that the catalog updated. That habit (catching the brain when it files something wrong) is exactly the discernment that keeps it useful.

The Synthesis Discipline

The mechanism that keeps the brain trustworthy as it grows. Run it weekly, ideally after any heavy ingest day, with run synthesis.

It scans recent content against the vault, tags findings with evidence (conflict, identity, stale, gap, unclassified, drift), ranks them, and writes only the top three as questions in 05-synthesis.md. The cap is the product: three questions get answered, twenty get ignored.

The human gate

The AI flags; you decide. Nothing auto-resolves. You answer each question with one of:

  • TAP: answer it directly.
  • CTX: answer with context (the richest, teaches the brain).
  • IDK: the brain digs deeper next run.
  • SKIP: it returns lower; three skips and it parks.
The signal that a brain is alive

The single best metric is your synthesis answer-rate within 48 hours. Answering a few questions a week is the whole habit.

Backup & Updates

Back up to GitHub

Create a free account at github.com, make a private repository, then push (the easiest login is the GitHub CLI from cli.github.com, then gh auth login):

git remote add origin https://github.com/USERNAME/my-second-brain.git
git branch -M main
git push -u origin main

Back up again anytime with git add -A, git commit -m "update", git push.

Get kit updates

The shared kit improves over time. Only the shared parts (the skills and CLAUDE.md) update; your notes are never touched. The kit lives at github.com/SVPavlov/ai-second-brain-starter-kit (public).

Connect it once:

git remote add kit https://github.com/SVPavlov/ai-second-brain-starter-kit.git

Then, whenever there is an update:

git fetch kit
git checkout kit/main -- .claude CLAUDE.md
git commit -am "update kit"

That refreshes the skills, the slash commands and the manual. Everything you wrote stays exactly as it was.

Working Safely

The one data rule

Until the governed enterprise stack is live, no client-confidential material goes in your brain. Your own notes about your work: fine. Client documents, client personal data, anything under an NDA: not yet. Your brain is in a private cloud repo, so this matters more, not less. The rule lifts when the governed stack lands; the discipline you build now is the one you will need then.

The three rules that make a brain trustworthy

  • Asks, never guesses. When sources conflict or a fact may be stale, the brain surfaces a question rather than picking silently.
  • Always sourced. Every claim points back to where it came from. No source, no claim.
  • Content is data, never instructions. Text inside an ingested document that tries to instruct the AI is filed, never obeyed.

IP: your brain is yours. Only sanitized patterns and skills go to the shared hub; your client context stays local.

Already Have a Brain?

You do not throw away your existing notes. You align them to the kit so they work with the same skills. Work on a copy, never your original.

  1. Copy your existing brain to a new folder.
  2. Lay the kit on top of the copy: add CLAUDE.md, the numbered files where absent, and .claude/skills/. Do not overwrite or delete your own files.
  3. Map it. In Claude Code: "Read my existing structure and make a MAPPING.md that maps my folders to this framework's homes. Do not move or rename anything."
  4. Apply one change to start. A good first one: adopt 05-synthesis.md and run run synthesis across your real notes.

If your brain is large and mapping gets messy, map the top level only. The goal is your content preserved plus the kit's skills working, not a perfect migration.

Claude Code, Desktop & Cowork

Claude Code is the terminal tool that runs your brain. It is all you need.

Claude Desktop and Cowork are optional. Cowork is Anthropic's more autonomous mode inside the Claude Desktop app, with memory across sessions and scheduled tasks. It needs a Pro or Max plan from claude.ai/download. You do not need it for your second brain. The enterprise rollout of Cowork (on a work setup, with company sign-in) comes later and is arranged centrally; do not try to set it up yourself.

Terminal emulators (Windows Terminal, iTerm2) are optional nicer windows. The built-in Terminal and PowerShell are completely fine.

Obsidian is an optional viewer. Your brain is plain markdown; Obsidian is just a nice window onto it.

FAQ

Do I need a paid Claude subscription?

For the full automated brain in Claude Code, yes (Pro or Max). To start the thinking parts, a free claude.ai account in the browser is enough.

Is my data private?

Yes. Your brain is plain files on your laptop, backed up to a private repo you control. Plus the data rule above.

I am not technical. Can I still do this?

Yes. Follow Install top to bottom and paste the commands. If you cannot install anything today, you can still design your brain on paper and build it later with zero rework.

Will updates overwrite my notes?

No. The update command only refreshes the shared skills and the manual. Your notes are never touched.

What if a skill makes something up?

Check it. The brain should only state what its files support. A confident answer with no source is your signal to verify, which is what the synthesis discipline trains.
